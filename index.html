fireside chat

portable, client-side (that's you) encrypted chat

    What is this and how does it work?
    Fireside is a portable, client-side encrypted chat client. So what does that even mean?
    Portable: The inner-workings of Fireside are all stored inside this html file, not on a server. Therefore Fireside is open source! This means you can right-click this webpage, hit save-as and download this html.You now have a 100% functional chat client on your computer. All you need is to open the file in a browser (with an internet connection) and you're good to go. This means I do not need to to necessary host this website on my own paid server in order to distribute it. The html file can be sent to friends by any means.
    Client-side: Since all of the inner-workings of Fireside are loaded in the html file, locally, in your browser we say that is is client-side since all the work is on done by your computer. You are the client!
    Encrypted: This is at the heart of what Fireside does. Encryption protects your data from snooping third-parties.

    Before using Fireside with another person you should have decided, in private, on a secret passphrase. Anyone with this passphrase will have access to your chats so it is important you choose a unique passphrase...the longer and more random the better.

    When you enter your passphrase Fireside takes a hash of it. Cryptographic hashes have four main properties:
    • They take an input of any length and return an output string of fixed length.
    • Hashes are unique. No input string will create the same output as another input string.
    • Hashes are extremely sensitive to their input. A hash of a sentence and a hash of the same sentence with a period at the end will be completely different.
    • No information from the input can be extracted from the output hash.
    These properties make hashes excellent for protecting your passphrase and chat messages. The hash that Fireside uses is the SHA-256 hash. The hash of your passphrase will be the key to encrypting your data. The hash of your password is then hashed again. This second hash is the unique chat id of your chat. Your secret plaintext passphrase is now discarded.

    At this point Fireside connects to a service call FireBase and creates a data node with your chat id. This is where your encrypted messages will be kept. FireBase is a service that maintains and persists data, sort of like Dropbox. Since your chat id is a hash, no info of you or your passphrase is stored on the FireBase server.
    Next, Fireside asks you for your name, to know whose chats to show you are yours and whose to show as other people's. At this point FireBase sees if the data node that is your chat id has any data in it. If it does it sends you that data and Fireside decrypts is and write the messages to your screen.
